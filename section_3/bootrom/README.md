BootROM（启动只读存储器）是嵌入在设备硬件中的一小段固件，它在设备开机的第一时间就会执行。它比操作系统或引导加载程序（bootloader）更早运行。主要功能包括：

初始化硬件 —— 配置 CPU、内存和基本外设，为系统运行更复杂的软件做好准备。

验证并加载下一阶段程序 —— 通常会检查引导加载程序或操作系统的完整性，然后将控制权交给它们。

提供安全基础 —— 因为存储在只读存储器中，无法被轻易修改，所以它是可信赖的启动代码。

举例来说：

在智能手机上，BootROM 可能会验证固件签名，确保只能加载官方固件。

在 PC 上，BIOS 或 UEFI 的作用类似于 BootROM（不过 PC 上通常是可擦写的闪存）。

可以把 BootROM 理解为设备的“第一步启动程序”，它永久存在于硬件中，负责安全地开启整个系统。

ROM: 只读存储器

BootROM 不主动运行

BootROM 不初始化硬件（是 CPU 运行 BootROM 里的代码去初始化硬件）

BootROM 只是存储器

而 CPU 要从 BootROM 取指令，就必须给 BootROM 地址

PC 的本质是一个地址（下一个要执行的指令的位置）

BootROM 是 CPU 上电后执行的第一条指令的来源。

为什么 mem[PC] 一定有值？

因为 CPU 在 reset（复位）时，PC 会被硬件强制设置到一个固定地址，
而 BootROM 正好被映射在这个地址范围内。

因此：
✔ PC 一定落在 BootROM 所在的地址区间
✔ mem[PC] 一定有值
✔ CPU 一定可以取到第一条指令


bootrom 就是你的 CPU 上电后存指令的地方。
